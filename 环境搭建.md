# Lidar_nav2_ws CPU 实车环境搭建

[English Documentation](./Environment_Setup_EN.md)

本文只覆盖 `deploy/cpu-mid360-nav2` 分支的 CPU-only MID360 实车部署环境。启动脚本和 launch 参数详见：[CPU 实车部署说明](./docs/cpu_real_deployment.md)。

## 1. 目标环境

| 项目 | 要求 |
| --- | --- |
| 操作系统 | Ubuntu 22.04 |
| ROS 2 | Humble Hawksbill |
| 车载 PC | CPU-only，已按十一代 i5 无 GPU 场景收敛 |
| LiDAR | Livox MID360 |
| 底盘接口 | 订阅 `/cmd_vel`，类型 `geometry_msgs/msg/Twist` |

加载 ROS 2：

```bash
source /opt/ros/humble/setup.bash
printenv ROS_DISTRO
```

期望输出 `humble`。

## 2. 获取代码

```bash
cd ~
git clone <repo-url> Lidar_nav2_ws
cd Lidar_nav2_ws
git switch deploy/cpu-mid360-nav2
git submodule update --init --recursive
```

本分支保留两个根目录第三方依赖：

- `third_party/robin-map`：KISS-Matcher 依赖。
- `third_party/KISS-Matcher`：仅作为 C++ 库依赖使用，不进入 colcon 包扫描。

需要由 colcon 构建的第三方包 `small_gicp` 放在 `src/99_vendor/small_gicp`。

## 3. 安装基础依赖

```bash
sudo apt update
sudo apt install -y \
  git wget curl vim gnupg lsb-release software-properties-common \
  build-essential cmake ninja-build pkg-config \
  python3-pip python3-colcon-common-extensions python3-rosdep \
  libtbb-dev libboost-all-dev libomp-dev
```

初始化 `rosdep`：

```bash
sudo rosdep init
rosdep update
```

如果当前网络环境下 `rosdep update` 较慢，可使用 `rosdepc`：

```bash
sudo pip3 install rosdepc
sudo rosdepc init
rosdepc update
```

安装 ROS 依赖：

```bash
cd ~/Lidar_nav2_ws
source /opt/ros/humble/setup.bash
rosdep install --from-paths src --ignore-src --rosdistro humble -r -y
```

常用运行依赖可显式补装：

```bash
sudo apt install -y \
  ros-humble-navigation2 \
  ros-humble-nav2-bringup \
  ros-humble-slam-toolbox \
  ros-humble-pointcloud-to-laserscan \
  ros-humble-robot-state-publisher \
  ros-humble-tf2-tools
```

## 4. MID360 网络检查

编辑 Livox 配置：

```bash
vim src/livox_ros_driver2-master/config/MID360s_config.json
```

重点检查：

- `host_net_info` 中的主机 IP 与有线网卡 IP 一致。
- `lidar_configs` 中的 MID360s IP 与实物一致。
- 主机和 MID360s 在同一网段。

连通性检查：

```bash
ping <MID360_IP>
```

## 5. 构建

```bash
cd ~/Lidar_nav2_ws
source /opt/ros/humble/setup.bash
cd scripts
./build.sh
```

`scripts/build.sh` 会调用 `scripts/build_cpu_real.sh`，只构建 CPU 实车白名单包：

```text
small_gicp
livox_ros_driver2
fast_lio
lio_interface
sensor_scan_generation
gld_robot_description
me_nav2_bringup
pcd2pgm
small_gicp_relocalization
global_relocalization_kiss_matcher
```

构建完成后加载工作空间：

```bash
source ~/Lidar_nav2_ws/install/setup.bash
```

## 6. 快速验证

确认 colcon 默认只发现 CPU 实车包：

```bash
cd ~/Lidar_nav2_ws
colcon list --names-only
```

确认 launch 参数：

```bash
source install/setup.bash
ros2 launch me_nav2_bringup cpu_real_mapping.launch.py --show-args
ros2 launch me_nav2_bringup cpu_real_nav.launch.py --show-args
```

## 7. 实车运行

建图：

```bash
cd ~/Lidar_nav2_ws/scripts
./mapping_real.sh map_name:=site_a use_rviz:=true
```

保存 2D 地图：

```bash
./save_map.sh site_a
```

保存 FAST-LIO 3D PCD：

```bash
./save_pcd.sh
```

`save_pcd.sh` 不接收地图名参数；PCD 路径由启动建图时的 `map_name` 或 `prior_pcd_file` 决定，默认通常位于 `install/me_nav2_bringup/share/me_nav2_bringup/pcd/<map_name>.pcd`。

导航：

```bash
./nav2_real.sh map_name:=site_a relocalizer:=small_gicp
```

`nav2_real.sh` 会在加载 `install/setup.bash` 时临时关闭未绑定变量检查；导航 launch 先启动并激活 `map_server`，再延后启动 Livox、FAST-LIO、Nav2 和重定位链路。

未知起点：

```bash
./nav2_real.sh map_name:=site_a relocalizer:=kiss
```

运行中检查：

```bash
ros2 topic hz /livox/lidar
ros2 topic hz /livox/imu
ros2 topic hz /cloud_registered
ros2 topic hz /registered_scan
ros2 topic hz /scan
ros2 run tf2_ros tf2_echo map odom
ros2 topic echo /cmd_vel
```

## 8. 常见问题

### `./build_cpu_real.sh` 报 `bash\r`

现象：

```text
/usr/bin/env: 'bash\r': No such file or directory
```

原因是脚本被转换成 Windows CRLF 行尾。Linux 读取 shebang 时会把解释器名识别成 `bash\r`。

处理：

```bash
cd ~/Lidar_nav2_ws
sed -i 's/\r$//' scripts/*.sh
chmod +x scripts/*.sh
```

本分支已加入 `.gitattributes`，后续从 Git 正常检出时脚本会保持 LF 行尾。

### `livox_ros_driver2` 报 `std::clamp`

现象：

```text
error: 'clamp' is not a member of 'std'
```

原因是 `livox_ros_driver2` 默认按 C++14 编译，而 `std::clamp` 属于 C++17。CPU 分支已将该处改为 C++14 兼容写法。更新分支后重新构建：

```bash
cd ~/Lidar_nav2_ws/scripts
./build_cpu_real.sh
```

### 找不到 ROS 2 包

```bash
cd ~/Lidar_nav2_ws
source /opt/ros/humble/setup.bash
source install/setup.bash
ros2 pkg list | grep me_nav2_bringup
```

如果仍找不到，重新构建：

```bash
cd scripts
./build.sh
source ../install/setup.bash
```

### MID360 无数据

按顺序检查：

```bash
ping <MID360_IP>
ros2 topic hz /livox/lidar
ros2 topic hz /livox/imu
```

常见原因是主机网卡 IP、MID360s IP、`MID360s_config.json` 三者不一致。

### `/scan` 或 `/registered_scan` 无数据

```bash
ros2 topic hz /cloud_registered
ros2 topic hz /registered_scan
ros2 topic hz /scan
```

如果 `/cloud_registered` 无数据，优先检查 Livox 驱动和 FAST-LIO；如果只有 `/scan` 无数据，检查 `vehicle.yaml` 中点云切片高度。

### Nav2 不输出 `/cmd_vel`

```bash
ros2 run tf2_ros tf2_echo map odom
ros2 run tf2_ros tf2_echo odom base_footprint
ros2 topic hz /scan
ros2 topic echo /cmd_vel
```

常见原因：

- `map -> odom` 未发布，重定位未成功。
- `odom -> base_footprint` 未发布，LIO 或桥接异常。
- 2D map 与 PCD 不是同一次建图结果。
- 底盘安全状态未解除，或底盘未订阅 `/cmd_vel`。
